# form表单提交

# form表单的常用属性

1、action：url地址，服务器接收表单数据的地址

2、method：提交服务器的http的方法，一般是get/post

3、name：表单属性的唯一性

4、enctype：表单数据提交时使用的编码类型，默认使用pplication/x-www-form-urlencoded，如果是使用post请求，则请求头中的content-type指定值就是该值。

# 浏览器提交form表单时，会执行如下步骤

1、识别出表单中表单元素的有效项作为提交

2、构建一个表单数据集

3、根据form表单中的enctype属性的值作为content-type，对数据进行编码

4、根据form表达中的action属性和method属性向指定地址发送数据

# form表单提交的四种数据格式

1、application/x-www-form-urlencoded

这种是最常见的post数据请求方式。浏览器的原生form表单，如果不设置form的enctype属性，那么最终请求的时候会以application/x-www-form-urlencoded的方式提交数据，请求格式是键对值（key1=val1&key2=val2）

<form id="formCon" name="formCon" method="post" action="xxxxx">
    UserName:<input type="text" name="username">
    Password:<input type="password" name="password">
    <input type="submit">
</form>

2、multipart/form-data

这种是上传文件时表单的提交方式，必须要让form表单的enctype属性等于multipart/form-data。

看一个例子

<form action="xxxx" method="post" enctype="multipart/form-data">
    UserName:<input type="text" name="username">
    Password:<input type="password" name="password">
    File:<input type="file" name="file">
    <input type="submit">
</form>

3、application/json

content-type为application/json响应头肯定不陌生，现在越来越多的开发用它作为请求头，用来告诉服务端消息主体是序列化后的json字符串，由于json规范的流行，除了低版本IE之外的各大浏览器都原生支持JSON.stringify，服务端语言也都有处理JSON的函数。

Google的AngularJs中的Ajax功能，默认就是提交JSON字符串。

4、text/plain

它是一种使用http作为传输协议，XML作为编码方式的远程调用规范。text/plain是以纯文本格式（也就是一段字符串）发送的，如果你发送一个对象例如{name:'xiaoMing',age:'12'}，那么一定要对它做JSON.stringify处理，否则将传送[object,object]。
