# margin踩坑总结

# 外边距合并

外边距合并是指块级元素的上外边距和下外边距有时会合并（或折叠）为一个外边距，其大小取其中的最大者。

例如：上外边距、上下边距和下外边距的合并

样式代码如下：

<div style="border:1px solid #ccc">
    <div class="bottom">
        <p style="background:rgb(173, 235, 206);margin:20px;text-align: center">margin：20px</p>
        <p style="background:rgb(173, 179, 235);margin:20px;text-align: center">margin：20px</p>
    </div>
</div>

其中涉及到的概念margin击穿：在上面这个例子中，给P标签设置的margin按理来说是要作用在bottom这个div上的，但是此时作用在了最外层的div，这就是margin击穿现象。margin击穿是指给子元素设置的margin却作用在了父元素身上。

# 相邻元素margin合并的解决方法

1、给外层div（bottom）加上border属性；

2、给外层div加上overflow：auto；

3、给外层div加上padding；

4、给外层div里面加上内容。

# 由margin合并拓展出的margin-right失效问题

一般情况：浏览器默认从上往下从左往右渲染元素，在没有超出父容器的宽度的前提下，如果子容器的宽度能够被容纳，设置margin-right就是没有用的。

下面请看一个特殊例子：

由下图展示可以看到，红色盒子设置的上、下、左边距都正常显示，但是右边距失效了。

CSS样式如下：

.father-box {
    overflow: auto;

    margin-top: 30px;
    width: 400px;
    height: 200px;

    background: #333;
}
.child-box {
    margin: 20px;
    width: 600px;
    height: 200px;

    background: orangered;
}

我们来分析一下，由上面说的浏览器渲染元素的原理可以知道，在这个例子中子元素的宽度大于大于父元素，所以它的margin-right就是有用的。又因为设置了overflow属性的元素和它的子元素之间的margin不被重叠（overflow取值为visible除外），所以此时又可以得出margin合并问题失效了。那究竟是什么原因导致的呢？下面就要了解一下关于BFC的知识。

官方文档是这样介绍的： 块格式化上下文（Block Formatting Context，BFC） 是Web页面的可视化CSS渲染的一部分，是块盒子的布局过程发生的区域，也是浮动元素与其他元素交互的区域。

回到这个例子中，可以看到父元素的样式中设置了overflow：auto的属性，那么就激活了父元素的BFC属性，此时父子元素处于同一个BFC中，又由于子元素的宽度大于父元素的宽度，所以导致了margin-right失效。

根据以上的思路，解决这个问题的方法就是让父子元素处于两个BFC中，激活子元素的BFC，所以给子元素设置display：inline-block，这个时候它们两个设置的样式互不影响，所以margin-right生效了。

以上的分析仅供参考，目前还没有官方答案。
